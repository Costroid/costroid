# Costroid тАФ Architecture & Build Spec (canonical)

The canonical **technical** source of truth for the build. This lives in the repo as `ARCHITECTURE.md` and governs every technical decision below it. It supersedes the earlier strategy docs for build purposes; launch/marketing material stays separate. **Scope and sequencing now defer to [PRODUCT-PLAN.md](PRODUCT-PLAN.md)** тАФ the going-forward, step-by-step production plan (the taskbar, the connections subsystem, the generalized quota model, and the two deferred providers all live there); this doc stays the technical authority and is trued up to that plan. Code-shaped notes describe intent and naming, not frozen signatures. Detailed companion specs (DATA-MODEL.md, DESIGN-SYSTEM.md) are referenced where they own the granular truth.

## 1. What Costroid is (and is not)

**Position:** the local, private FinOps tool for AI coding assistants тАФ the one that *also* sees the subscription limits the platforms can't.

**It is NOT:** a platform, observability, enterprise FinOps, a proxy/gateway, a chatbot or conversational-LLM interface, or anything that processes prompt/completion content. **Metadata only.** (Any web platform is a separate, separately-licensed repo тАФ never built here.)

**Competitive frame:** measured against **ccusage** and the status-bar tools тАФ not Finout, OpenCost, or Vantage. The two load-bearing differentiators, because no billing-based competitor can do them: **subscription-quota awareness** (no invoice exists for it) and **local-first / no content / zero integration**.

**Audience:** individual developers, for now.

## 2. The intellectual core: the costтАУquality frontier

Price and quality don't track each other, and that gap is why this product exists.

- DeepSWE (avg cost/task, updated 2026-05-30): gpt-5.5 = 70% @ $6.61 vs claude-opus-4.7 = 54% @ $18.19 тАФ gpt-5.5 cheaper *and* better; claude-sonnet-4.6 = 32% @ $5.52 (cheapest тЖТ on-frontier). DeepSWE's cost column is cache-miss-priced (~5├Ч, indicative). opus-4.8 isn't benchmarked here yet тЖТ show as a gap, never a guessed number (┬з9.3).
- CursorBench v3.1 (2026-05-18): Composer 2.5 = 63.2% @ $0.55 vs Opus 4.7 Max = 64.8% @ $11.02 (matched at ~1/20th the cost); gpt-5.5 = 64.3% @ $4.37. Vendor benchmark (Cursor's own private tasks, non-reproducible, showcases Composer) тАФ corroborating only; its **cost is cache-aware** (computed from published per-token pricing), so unlike DeepSWE the cost is sound тАФ the caveat here is provenance, not cost.

The frontier is jagged: the cheapest model is sometimes a trap, the priciest often isn't best. So the product **shows the frontier, plots where the user's spend sits on it, and lets them judge** тАФ never "use the cheapest." It takes model *quality* from the benchmarks but the *dollar* axis from Costroid's own cache-correct cost (a benchmark task-average isn't the user's actual bill; ┬з12 has the per-benchmark cost caveats). It stays honest that it sees spend and benchmarks but *not* task difficulty, so it informs rather than prescribes. That honesty is what makes it trustable, and this principle governs the whole UI тАФ not just one screen.

## 3. Scope (v1)

*Current state: this v1 core is built and **shipped through v0.3.0** (with the T7/T8 connect groundwork since landed on `main`); the live "where are we" is [PRODUCT-PLAN.md](PRODUCT-PLAN.md) ┬з0. The list below is the v1 scope as designed.*

- **Providers:** Claude Code + Codex (validated), Cursor (v1: **detection only** тАФ present + selected model, labeled beta; its usage/quota are served live server-side with no sanctioned source, so they stay **"unavailable"** тАФ discovery-gated, ┬з8, never via session reuse).
- **Cost:** subscription quota (%) *and* API cost ($); filter by provider. Cross-provider totals are **per-lane** (see ┬з6 тАФ API spend, subscription-equivalent value, and quota % shown distinctly), never one merged number.
- **Trends dimensions:** aggregate by period (`day` / `week` / `month` / `year`) and group (`model` / `app` / `total`).
- **Export:** FOCUS records to JSON and CSV.
- **Surfaces:** the TUI (`now` / `trends`, with the recommendation/frontier surface reached via the `a` ask key + the insight line); a `costroid statusline` mode (tmux, Starship, Claude Code's `statusLine`); and **`costroid setup-statusline`** тАФ a first-class onboarding command that wires Claude Code's `statusLine` (the capture snippet into an existing one, or Costroid as the statusline) so live Claude quota works. A cross-platform **taskbar/tray app is planned** тАФ the last surface (PRODUCT-PLAN.md Step 6, target v0.6.0), built in **egui / eframe + the `tray-icon` crate** (all MIT/Apache-2.0; no Tauri, no webview) as package `apps/bar` (binary `costroid-bar`); it consumes the core every other surface already shows.
- **Recommendation:** a frontier view from **DeepSWE (primary)** + **CursorBench (corroborating)**, as a bundled snapshot.

## 4. Technical canon (constraints the build inherits)

- Rust; single Cargo workspace; edition 2021; **MSRV 1.88** (documented in CI); workspace crates versioned **in lockstep** (`[workspace.package].version`); commit `Cargo.lock`. **CI gate (green = releasable):** `rustfmt` + `clippy -D warnings` + `cargo test` + FOCUS-conformance + `cargo deny` (license + advisories) + offline-acceptance (no-network).
- `thiserror` in libs (`ProviderError`/`FocusError`/`CoreError`), `anyhow` in the binary. **No `unwrap`/`expect`/`panic` in libs** (tests use the `match тАж panic!` idiom).
- `tracing` with an env filter, **local diagnostics only, never networked**; quiet by default, `-v`/`-vv` to stderr (**planned convention тАФ not wired yet:** nothing links `tracing` today and the CLI defines no verbosity flags; adopt this shape when diagnostics are first needed). **No telemetry.** License **Apache-2.0**; dependencies must be **permissively licensed** (MIT / Apache-2.0 / BSD / ISC / Zlib / Unicode) тАФ **no copyleft** (GPL / AGPL / LGPL / SSPL); verify before adding (the CI gate runs `cargo deny`).
- Standards: FinOps Foundation + **FOCUS 1.3** (validator `finopsfoundation/focus_validator`, run offline against the **vendored 1.3.0.1 ruleset** at `scripts/focus-ruleset/` тАФ the PyPI wheel bundles only 1.2.0.1, so a `--block-download` 1.3 run needs the vendored model; 3 known validator-ruleset defects allowlisted with upstream refs тАФ only the rules that fire, never their passing siblings). Treat the FOCUS spec, not the docs, as the authority on column semantics.
- Pricing **bundled inside `costroid-core` via `include_str!`** at `crates/costroid-core/pricing/pricing.vN.json` тАФ **cargo packages only files under the crate dir, so all bundled data must live inside its crate** (a workspace-root location breaks `cargo publish`). Sourced current at build time and refreshed deliberately тАФ never hardcoded figures that drift; works fully offline. Records `as_of` + `sources`; rates per meter (input/output/cache_read/cache_write) **per 1M tokens**; the calculator derives the **per-token** unit price (rate ├╖ 1e6). Catalog key = model id (e.g. `claude-sonnet-4-6`), not display name. Cost is always labeled an **estimate**; reconciliation against the real invoice is Phase 2+.
- **Config (planned convention тАФ not wired yet):** TOML at the XDG path (`~/.config/costroid/config.toml`) with sensible zero-config defaults; no config system is built today тАФ everything runs zero-config on built-in consts. **Secrets never go in config тАФ keychain only.**
- crates.io: **iterate the existing names** (`costroid`, `costroid-core`, `costroid-focus`, `costroid-providers`) тАФ published through 0.3.0; the next iteration is 0.4.0, which extends the publish ladder with `costroid-connect` (T10b, RELEASING.md) тАФ don't orphan or rename them. npm: `costroid`.
- **Brand:** monochrome (black/grey/white, adapting to the terminal theme) + a **single amber accent reserved for the warning/near-limit state** (red only as an intensification, always with a non-color cue); JetBrains Mono; braille glyphs (U+2800 + bitmask); **mandatory `--plain` fallback** (linear labeled text, no color, no braille тАФ ┬з7). Mark `CтаЙ` (a pixel `C` beside the braille cell `таЙ`); wordmark `costroid` with `cost` in the strong weight and `roid` muted тАФ carry the split into the UI: **dollar figures and the active metric use the strong weight; labels and context are muted.** (Dot math + component specs in DESIGN-SYSTEM.)
- **Log discovery (WSL-aware):** Claude Code = `~/.claude/projects/**/*.jsonl` + `~/.config/claude/projects/**/*.jsonl`; Codex = `~/.codex/sessions/**/*.jsonl`; Cursor = **no local usage/quota** тАФ Cursor serves these live from its own backend (not to local logs), and Costroid does **not** call Cursor's internal endpoints, so Phase 1 only **detects presence + selected model** from the `~/.cursor` config (honor `CURSOR_DATA_DIR`), never chat content. **Honor `CLAUDE_CONFIG_DIR` (comma-separated roots) and `CODEX_HOME` before the defaults; merge all roots.** Detect WSL via `/proc/sys/kernel/osrelease` (contains `microsoft`/`WSL`); under WSL, **scan `/mnt/c/Users/*` for the profile(s) that actually hold the logs** (so a WSL user тЙа Windows profile, e.g. `eren`/`ereno`, still resolves), with `USERPROFILE` (if set) and the legacy `$USER` path as fallbacks. Native roots are XDG / `~/Library` / `%APPDATA%`. Never assume a single fixed path. Claude Code keeps **no quota in its session logs**, but its live 5h/7d limits arrive *locally* through its **`statusLine` hook** тАФ the `rate_limits` block (`five_hour`/`seven_day` тЖТ `used_percentage` 0тАУ100 + `resets_at` epoch s; Pro/Max, after the first API response; **zero API tokens**), captured by the statusline shim (┬з7/┬з8), so **no credential-reuse endpoint is needed for Claude**. Codex exposes its rate-limit windows in the session logs directly (5h = 300 min, weekly = 10080 min).

## 5. Workspace architecture

- **`costroid-core`** тАФ cost calc + FOCUS normalization orchestration + bundled pricing + a **`bench`/`recommend` module** (the DeepSWE/CursorBench snapshot, the reconciliation rule, the frontier computation). No terminal/UI code. *Ported and test-guarded.* The trust foundation. Promote `bench` to its own crate only if it grows.
- **`costroid-focus`** тАФ the FOCUS record types and their (de)serialization (JSON/CSV). Pure data, **no internal dependencies**; the crate most likely extracted/reused later. *Ported.*
- **`costroid-providers`** тАФ the provider trait + one adapter per provider + WSL-aware discovery. No internal dependencies (it emits the provider-neutral `UsageEvent`/`LimitWindow`; FOCUS normalization lives in `costroid-core` тАФ the once-declared `costroid-focus` dep was unused and removed in the 2026-06-10 fix pass). Claude + Codex *ported/validated*; Cursor *new (beta)*. Live quota enters via **sanctioned/local sources only** тАФ the Claude `statusLine` push and Codex's local windows (┬з4/┬з8); Cursor has no such source, so its live quota is **"unavailable"** (discovery-gated тАФ pursued only if Cursor ships a sanctioned API; never via session reuse, ┬з8). Costroid never reuses credentials against undocumented endpoints.
- The binary crate is **`costroid`** (the `cargo install costroid` / `npx costroid` name) тАФ clap CLI + the Ratatui TUI + the statusline + `--live` + the `--plain` path. It lives at **`apps/cli`** (package `costroid`, binary `costroid`).
- `fixtures/` holds committed sample provider logs for tests тАФ **never real user data**.
- **`costroid-connect`** тАФ all network + credential code, feature-gated and **off by default**. The crate is feature-gated (skeleton landed in T7); its **keychain behavior landed in T8** тАФ `keyring` 3.6.3 (sync Secret Service on Linux via C libdbus, Apple/Windows native backends) providing `CredentialStore` + a non-secret `ConnectionRegistry` + the `ApiVendor` enum, with **no `core`/`focus` dependency yet** тАФ and the network half (`ureq` + `rustls`, no async runtime) lands in T9, all at Step 4 (v0.4.0). The sync backend pulls **no async runtime**, so `async-io`/`tokio` stay forbidden even under `--features connect`. The `connect` feature is gated on the **apps** (`apps/cli` today, `apps/bar` later), off by default тАФ a virtual workspace root holds no `[features]`. **Still planned (not yet in the workspace):** **`apps/bar`** (binary `costroid-bar`, egui/eframe + `tray-icon`, depends only on `costroid-core`) тАФ the taskbar surface at Step 6 (v0.6.0). No `costroid-mcp` in any scope (the crates.io name stays intentionally unclaimed; the recommendation engine it would have exposed is built into the `bench` module).

**Dependency direction (no cycles):** `costroid-focus` and `costroid-providers` sit at the bottom and depend on nothing internal; `costroid-core тЖТ providers + focus`; the `costroid` binary тЖТ core; nothing depends on the binary. When the `connect` feature is on, the binary also тЖТ `costroid-connect` тЖТ {core, focus} (the app gates it; `costroid-connect` publishes after `costroid-core`, per RELEASING.md) тАФ still acyclic; this `core`/`focus` edge is dormant until `costroid-connect` gains its network/reconciliation behavior in T9 (T8's keychain credential store deliberately added no `core`/`focus` dependency).

**Data flow (the default / local-only build makes no network calls тАФ still enforced by the strace offline-acceptance test + the forbidden-crates test):** `discover()` тЖТ `parse` usage + limits тЖТ normalize to FOCUS + estimate cost тЖТ aggregate (period, group) тЖТ render (now/trends/statusline) **or** export (JSON/CSV). Renderers and exporters are pure consumers of the same in-memory model. **Network happens only through `costroid-connect`** (the crate landed its off-by-default skeleton in T7 and its keychain credential store in T8; its network behavior lands at Step 4 (T9)), behind a Cargo feature **and** an explicit, user-initiated `connect` action to a provider endpoint the user authorized. The static offline test (`apps/cli/tests/offline.rs`) was **already re-scoped in T7** into a two-tier resolved-graph check тАФ the default build forbids all networking/TLS/telemetry (incl. the `ureq`/`rustls`/`keyring` trio) and asserts `costroid-connect` is not linked; `--features connect` admits only that trio and asserts it is linked тАФ and the dynamic strace acceptance script grew a feature-on baseline in T8 (the `--features connect` build leaks no network and writes no `$HOME` residue on a normal run), with the feature-on connect-action network test still stubbed for T9/T10 (the HTTP client + the `connect` CLI). No telemetry, ever.

**Provider trait semantics:** `discover()` returns "no data" (not an error) when a provider isn't installed тАФ missing providers are skipped, not fatal. `costroid-core` holds a registry of providers; the CLI runs all detected or a subset via flags. Adding a provider should require **no changes outside `costroid-providers`** (the test of the abstraction). Limits come from local data in Phase 1 тАФ including Claude's live 5h/7d via the sanctioned `statusLine` push (┬з8) and Codex's local windows; a provider with no sanctioned source (Cursor) stays detect-only, its quota "unavailable" (┬з8) тАФ Costroid never reuses credentials to fetch it. **Three providers ship today (Claude Code, Codex, Cursor).** **GitHub Copilot** and the **Antigravity CLI** are *planned* тАФ added via the **`Capability` descriptor** тАФ now a required `capability()` method on the `Provider` trait, landed in T3 (PRODUCT-PLAN.md ┬з2b, ┬з8) тАФ but **only after a live-install discovery confirms each one's real data/auth/quota shape; never build either adapter speculatively.**

## 6. Data model (summary тАФ full spec in DATA-MODEL.md)

- **FOCUS output.** Each unit of API usage is emitted as FOCUS 1.3 Cost & Usage rows with `ChargeCategory = "Usage"`. Input / output / cache-read / cache-write each become a **separate row** (distinct `SkuId`/`SkuPriceId` + `x_TokenType`) so quantities and unit prices stay coherent. Use the active 1.3 participating-entity columns **`ServiceProviderName` / `HostProviderName` / `InvoiceIssuerName`**; the deprecated `ProviderName` / `PublisherName` are **also emitted** (mirroring the active values) only because the bundled 1.3 validator requires their presence тАФ drop them with a 1.4 ruleset. `ServiceCategory = "AI and Machine Learning"`. Unit prices are **per token**; `PricingUnit = "tokens"`.
- **Custom `x_` columns:** `x_Model`, `x_TokenType` (input|output|cache_read|cache_write), `x_AccessPath` (api|subscription|unknown), `x_Estimated`, `x_PricingStatus` (priced|missing_price|unknown_model), `x_Tool` (claude-code|codex|cursor), `x_Project`, and **`x_ConsumedTokens`** тАФ the raw token count, **always populated** (the aggregation engine totals from this).
- **Subscription limits are a separate type, not FOCUS rows.** A `LimitWindow` carries `tool`, optional `plan`, `kind` (`FiveHour` | `Weekly` | `Daily` | `Monthly` | `BillingCycle`), `measure: Option<LimitMeasure>` (a `TokenFraction` (0тАУ1) **or** `Spend { used_usd, included_usd }` тАФ a request-count measure is the legacy pre-June-2026 Copilot model, intentionally **not** implemented), `resets_at`, plus `captured_at: DateTime<Utc>` + `status: LimitStatus { Verified, Unverified, Unavailable }` тАФ **no dollars**, never summed into a bill. **This generalized shape landed in T2** (PRODUCT-PLAN.md ┬з2a / ┬з3 T2), replacing the old `used_fraction`. The core `LimitAvailability` separately gains `Unverified` + `Estimated` variants тАФ **at the availability/render layer only, never on the provider `LimitWindow`**. Their **live producers landed in T4** (the cross-check demote `VerifiedтЖТUnverified`, the stale age-out, and the volume-based `Estimated` fallback); the **rendering of these states landed in T6** (the now-screen + statusline render all five arms тАФ `Unverified` carries a `? unverified` cue + neutral meter, `Estimated` shows volume + `~$value` with no meter, `Spend` shows `$used / $included used`).
- **Three lanes, never summed across each other:** (1) **API usage** тЖТ FOCUS rows, real cost estimate, `x_AccessPath = "api"`; (2) **subscription usage** тЖТ FOCUS rows valued at **API-equivalent** (`x_Estimated = true`, `x_AccessPath = "subscription"`) тАФ labeled estimated value, **never a bill**; (3) **subscription quota** тЖТ `LimitWindow`s (%). The cross-provider "total" is therefore **per-lane** (API spend, subscription-equivalent value, quota % shown distinctly). Recommendations attach **only** to `x_AccessPath = "api"` rows.
- **Access path is detected from evidence, never guessed.** Codex: `rate_limits` windows present тЗТ subscription. Claude Code: from auth mode via non-secret **presence** signals only (never read credential values). `unknown` when there's no signal.
- **Unpriced rows** (no matching rate, or empty pricing table): the four cost columns stay present and `0`, `SkuPriceId` is null, and the FOCUS-required pricing/consumption columns go null accordingly тАФ but `x_ConsumedTokens` still carries the count, so unpriced usage is **never dropped** from totals; flag `x_PricingStatus`. **Never substitute a guessed price.**
- **Export contract:** JSON is a wrapper `{ "focusVersion": "1.3", "rows": [...] }` (never a bare array); CSV uses the exact FOCUS PascalCase header (`x_` columns appended). `LimitWindow`s export **separately**, never mixed into the cost data.
- **Grouping:** by `x_Model`, by `x_Project` (bucket `"unknown"` when undeterminable), or `total`; period buckets by `ChargePeriodStart` in the user's local time zone. Never sum `LimitWindow` data.

DATA-MODEL.md remains authoritative for the column-by-column mapping, the `UsageEvent`/`FocusRecord`/`LimitWindow` Rust shapes, per-provider field paths, the pricing JSON schema, and the FOCUS-validator nullability/defect mechanics.

## 7. Rendering & UX (mechanics here; dot math + mockups in DESIGN-SYSTEM)

- **Ratatui + crossterm.** Braille is computed directly from `U+2800` + an 8-dot bitmask тАФ **not** Ratatui's braille constants or `Canvas` (those drift across versions, and `Canvas` is TUI-only). The **sparkline is hand-rasterized to braille** like every other component, so the one-shot and TUI paths stay identical (see the styled-document rule below).
- **Fill = bright vs dim cells.** A terminal cell has a single foreground color, so braille gives sub-cell *shape* but not sub-cell *color*: used vs remaining is **bright vs dim cells** at cell granularity (the solid-dot/hollow-ring look from the mockups is raster-only тАФ website/marketing). **Under `NO_COLOR`, only ANSI styling is dropped тАФ braille meters remain** and stay readable because used vs remaining are *glyph-distinct* (full `тг┐` vs the `тгА` track), never color-alone; the ASCII `[####--]` rendering is reserved for braille-incapable terminals (`RenderMode::Ascii` тАФ the fully-ASCII *visual* fallback, same layout with ASCII meters).
- **Warning thresholds:** `warn = 0.80`, `critical = 0.95` (defaults, configurable). The warning state turns amber (red at critical/over) **and always carries a textual cue** тАФ `!`, `!!`, `OVER`, or "near limit" тАФ so it survives `NO_COLOR`, color-blindness, and `--plain`. Cost bars never go amber (amber is for limits, not spend).
- **`RenderMode`:** `Braille` (default) / `Ascii` (the fully-ASCII visual fallback when braille is unsupported тАФ internal, not a user flag) / `Plain` (`--plain`: no TUI chrome, no color, no braille, no meters тАФ linear labeled text, screen-reader friendly; Costroid-generated text in it is pure ASCII, pinned by test, while provider-supplied names pass through verbatim). Mode is chosen from `--plain`, TTY detection (non-tty тЗТ Plain), `NO_COLOR`, and a braille-capability check. The only user-facing mode flag is `--plain`.
- **Interactive vs one-shot:** on a TTY, `now`/`trends` run as a navigable TUI; piped output, `--plain`, `statusline`, and `export` render once and exit.
- **One styled document, two adapters:** the renderer emits a neutral styled document (semantic styles: strong/dim/warn/critical/plain). A one-shot adapter serializes it to an ANSI/plain string; the TUI adapter maps it to Ratatui тАФ keeping both identical. The one-shot serializer is snapshot-tested as the compatibility contract.
- **Terminal is always restored** (raw mode off, alternate screen left, cursor shown) on quit, error, and panic тАФ via a restore guard plus a panic hook that leaves the alternate screen, chains the default handler, and exits 101. Works over SSH and inside tmux.
- **`--live`** re-collects data periodically (~2s); without it the screen is a snapshot refreshed manually (`r`).
- **`costroid statusline`** prints a single compact line (shell prompts, tmux, Starship) from the same core data and honors `RenderMode`. It is fast, and side-effect-free **on interactive stdin** (tmux/Starship тАФ stdin is never read); with **piped** stdin (Claude Code's `statusLine` JSON) it opportunistically captures the `rate_limits` block into the local no-secret cache before rendering (path 2, T5 тАФ that capture is the point of the surface). It also supports `--capture-only` (write the cache and exit) and `--wrap '<cmd>'` (capture, then delegate to an existing status-line command); all three paths degrade to a blank line + exit 0 on failure, never breaking the user's prompt. A configurable format string / presets is a planned addition, not yet built.
- **Screens.** `now` = stacked 5-hour + weekly **limit meters with reset countdowns**, then **API spend by model** (cost bars), then one insight line; limits and costs are visually parallel but clearly separate (limits carry no dollars). `trends` = a period **sparkline**, then the **breakdown cost bars**, then one insight line.
- **Keybindings:** `d`/`w`/`m`/`y` set period; `g` cycles group; `tab` switches screen; `f` or `/` filters (fuzzy model/app); `a` asks (hands context to the recommendation view, ┬з10 step 4); `r` refreshes; `q`/`Ctrl-C` quits (always restoring the terminal); `?` help.
- **States:** *loading* = braille spinner + short label; *empty* = plain-language note on what Costroid looked for and how to point it at the data (incl. the WSL/Windows path note), never an error; *partial* = show what's available and **label the gap explicitly, never fabricate** (e.g. Cursor); *per-provider error* = inline, non-fatal, the rest still renders.

## 8. Security boundary & credentials

The whole trust story depends on this.

- **Secrets live only in the OS keychain** (`keyring` crate: macOS Keychain / Windows Credential Manager / Linux Secret Service). **Never written to disk, config, or logs.**
- **Credentials flow only between the device and the provider** тАФ there is no Costroid backend or server.
- **Sanctioned channels first тАФ Costroid does not reuse a provider's credentials when a provider-built or local source exists.** Both validated providers are covered *inside the Phase-1 trust envelope* (no Costroid network call, no credential touched): **Claude Code's `statusLine` hook** тАФ an extension point Anthropic builds and documents for third-party scripts тАФ pushes a live `rate_limits` block (`five_hour`/`seven_day`, each `used_percentage` 0тАУ100 + `resets_at` epoch s; Pro/Max only, after the first API response; **runs locally, zero API tokens**); and **Codex's session logs** already carry the 5h/weekly windows (┬з4). Costroid captures the Claude numbers without a subprocess where possible: the lower-friction path is a **one-line capture snippet the user adds to their existing statusline** (it `tee`s the `rate_limits` block into a no-secret local cache тАФ Anthropic's own documented caching pattern); for users without one, **Costroid *is* the statusline** (renders + captures natively). **The cache** is a JSON file at `${XDG_STATE_HOME:-$HOME/.local/state}/costroid/claude-rate-limits.json` (Linux-side only тАФ it is Costroid's own state) holding a top-level `captured_at` (RFC3339) plus `five_hour`/`seven_day` windows, each `used_percentage` (0тАУ100) + `resets_at` (epoch seconds **or** RFC3339); it holds **no secret**. **T4 reads + sanitizes it** (absent/unreadable тЖТ Unavailable, never an error); **T5 shipped the writer** тАФ `costroid setup-statusline` wires Claude Code's `statusLine` to capture into this cache (via `costroid statusline --capture-only`, or by Costroid *being* the status line), so the cache populates after the next Pro/Max response; **T6 renders it on the now-screen** (the Available/Unverified/Estimated arms with the always-on "as of HH:MM" freshness stamp and the claude.ai chat caveat sub-note). *Wrapping* another command is a hazardous fallback only (you must tee stdin or the wrapped command renders blank, and you stack latency inside Claude Code's status-line debounce budget). Never an Anthropic endpoint call with Claude Code's token тАФ the **opposite of the codexbar credential-reuse pattern**, and the canonical Claude-quota mechanism. **Coverage decision (Opus weekly).** The hook carries `five_hour`/`seven_day` only тАФ **no per-model weekly window** (Opus or Sonnet). So **lead with the overall 7d % as the real number Costroid can measure тАФ not as "your binding cap"**: for a ~97%-Opus user the **Opus weekly may bind first**, and that sub-cap's % is exactly what the hook can't see. **The Opus-specific render is design intent, not yet shipped** (STATUSLINE-CAPTURE-BRIEF ┬з6's `opus_weekly` field): the design shows Opus 7d **volume + estimated value from local logs** (itself a local-log approximation of Anthropic's window, not the window) beside the 7d %, marks the Opus cap-relative **% unavailable**, and **discloses that a near-limit alert on the 7d window may not be the window actually throttling an Opus-heavy user** тАФ but no `opus_weekly` field or disclosure string exists in code. What the code does today: the now screen renders the **overall** 5h/7d windows (with the claude.ai chat caveat) plus the generic per-model calendar-week subscription-value rows in the cost section тАФ Opus appears there like any other model, never with a fabricated cap-%. Only the OAuth endpoint exposes the Opus %, and that path is dropped (the single strongest reason it stays documented-but-unused, ┬з12). Never fabricate an Opus %.
- **Costroid never reuses a stored credential, session, or token to hit a provider's undocumented or non-sanctioned endpoints** тАФ that is the ToS line, with no exceptions. This includes Cursor, whose usage/quota are served live server-side only (no local data, no hook); with no sanctioned source, Cursor stays **detect-only** and its cost/quota are **"unavailable," never fetched** (live quota is discovery-gated тАФ PRODUCT-PLAN.md ┬з8). **Auth source ladder** тАФ tiers 0тАУ3 are the only ones ever built (preference-ordered; PRODUCT-PLAN.md ┬з5 owns the per-provider classification): **(0) local artifacts** тАФ logs/config, no login, no credential, no network (Claude session logs, Codex logs, Cursor presence); **(1) sanctioned push/hook** тАФ Claude's `statusLine` `rate_limits` block, still zero network and zero credential; **(2) sanctioned OAuth** тАФ first-party where the provider documents it (e.g. GitHub; deferred), system browser + loopback redirect with PKCE (`oauth2`), token stored to the keychain only; **(3) your own API key** тАФ the provider's official usage/billing API (Anthropic / OpenAI / Gemini), your key, pasted by you and held in the keychain, used to reconcile the local estimate (Step 4); **(4) NEVER reuse any credential, session, or token against a non-sanctioned, undocumented, or internal endpoint** (this includes Cursor's `api2.cursor.sh` RPC) тАФ that is the account-ban path and a ToS violation, so the datum stays "unavailable." **No browser-cookie reading.** Even tier (2)/(3) can't expose Anthropic *subscription* 5h/7d limits (no sanctioned API does) тАФ the `statusLine` hook is the only route, which is why it's canonical.
- TLS via **rustls** (no OpenSSL). Network access is confined to provider endpoints the user explicitly authorized.
- **Provider logs are untrusted input** тАФ parsed defensively; malformed data yields a clean error or "unavailable", never a crash, and the parser never executes or evaluates anything from log content.
- A **connections view** (planned, PRODUCT-PLAN.md Step 4) lists what's linked and supports instant disconnect/revoke (`costroid connect`/`disconnect <provider>`); all network + credential code lives only in `costroid-connect`.

## 9. Non-negotiable principles (trust + don't over-engineer)

1. **The cost core is sacred.** The committed **fixture** golden tests vs. ccusage are the hard gate тАФ if a refactor moves a number, the build fails. Real-log ccusage parity is a one-off confidence check, not a bit-identical-forever mandate (ccusage can legitimately differ).
2. **Fragile parts degrade, never crash.** Any data source that is absent, malformed, or stale returns data *or* a clean "unavailable" тАФ never a crash. The Claude `statusLine` `rate_limits` field is **untrusted, and wrong in two distinct ways.** (a) *Out-of-range / poisoned:* a bug emits the epoch timestamp in `used_percentage` before the 5h window has data, and weekly has shown impossible values like 900% тАФ so a value `> 100` or equal to `resets_at` means *no data*; sanitize it out. (b) *In-range but wrong:* the same field has documented false readings like a flat **100% with no actual throttling** тАФ an in-range value no `>100` check can catch. Guard it with a **cheap divergence cross-check** (built in T4, core `finalize_limit_status`), not more sanitizing: if the statusLine % is high (тЙе `0.80`) but Costroid's own local token volume for that window is trivial (< `5_000` tokens, `UNVERIFIED_TOKEN_FLOOR`), **demote it to "unverified"** rather than rendering a confident "you're maxed." The cross-check can only *flag*, not correct тАФ a high reading may also be legitimately real (shared claude.ai-chat usage, or server-side fast-consumption) тАФ so the local estimate is a **validator when the field is present**, not just a fallback when it's absent. **Staleness, separately, is push-only and breaks in two directions:** *high across an idle reset* is **fixable** тАФ use `resets_at` to age a stale reading out to "unknown"; *low because quota was spent on another first-party surface* (claude.ai chat shares the same 5h/7d limit) is **not fixable, only disclosable** ("reflects only what Claude Code has observed; your true account usage may be higher"). Showing nothing or "unverified" is correct; showing a confident wrong number is fatal.
3. **Benchmarks supply quality, not cost.** They're a bundled, dated, cited snapshot тАФ not a live fetch. DeepSWE primary (a neutral data company; scored on the neutral `mini-swe-agent` harness, *not* the user's actual Claude Code / Codex / Cursor), CursorBench corroborating (Cursor's own vendor benchmark, showcases Composer). Take the **score** axis from them; take the **dollar** axis from Costroid's own cache-correct cost тАФ a benchmark task-average isn't the user's bill, and the published $/task vary in trustworthiness (DeepSWE's is cache-miss-priced ~5├Ч and disputed; CursorBench's is cache-aware but vendor-computed on private tasks тАФ ┬з12). Where they disagree or lack a model, show both and recommend only what's supported тАФ never invent a number. Surface source, date, and the harness/vendor caveat in the UI.
4. **Quota (%) and dollars ($) are different types.** The cross-provider total sums dollars only, per lane; quota shows alongside.
5. **Local, no content, no telemetry.** The brand and the trust both depend on it.
6. **The recommendation is a frontier view, not a prescriber.** It plots the published cost-vs-quality frontier and overlays the user's actual model mix and spend; it never claims "this task should have used X." It applies **only to API-cost rows** (model choice only changes the API bill), never subscription rows, and each recommendation carries its reasoning, its sources, and a projected dollar delta. All output is advisory.
7. **Tested against fixtures, not the network.** Unit tests per crate; integration tests driven by committed `fixtures/` logs (never real user data); `insta` snapshot tests for rendered output, especially `--plain`; and a no-network acceptance test that enforces the no-network rule. (CI gate is in ┬з4.)
8. **Speak like a colleague.** The insight line states the fact, then the so-what, then optionally a next step тАФ proactive, plain, specific, brief. **Hedge estimated costs** (`~`, "estimated", "about") тАФ never false precision on inferred money. One insight at a time, quiet by default, never blocking. Sentence case; no emoji, no fake urgency, no greetings, no chatbot register.

*Status (the live "where are we" lives in [PRODUCT-PLAN.md](PRODUCT-PLAN.md) ┬з0): the v1 CLI core is built тАФ the cost core verified to the cent vs ccusage, the frontier (`costroid frontier`) gate-green, plus WSL Windows-root auto-detect and Cursor detect-and-defer (┬з12). ## 11. Distribution & release

- **cargo-dist** (the installed binary is `dist`; invoke as `dist тАж` / `cargo dist тАж`); `dist init` writes the config + GitHub Actions workflow. **Nothing publishes on a normal push** тАФ only a pushed `vX.Y.Z` tag (which must equal `[workspace.package].version`) triggers CI to build and publish; PRs run `dist plan` only. **Targets (6):** Linux gnu x86_64/aarch64, Linux musl x86_64 (static), macOS x86_64/aarch64, Windows x86_64. (cargo-dist is actively maintained тАФ v0.32.0 тАФ by axodotdev; if it ever stalls, fall back to hand-written installers + `cargo-binstall`, or `release-plz`.)
- Channels: shell + PowerShell installers on GitHub Releases (`releases/latest/download/costroid-installer.sh | sh`); the Homebrew tap (`Costroid/homebrew-tap`, auto-generated by cargo-dist); the npm wrapper (`npx costroid` runs the native binary, no JS runtime); crates.io тАФ **published separately via `cargo publish` in dependency order: `costroid-focus` тЖТ `costroid-providers` тЖТ `costroid-core` тЖТ `costroid`** (see RELEASING.md), `cargo install costroid` / `cargo binstall costroid`. The `costroid-mcp` name is intentionally left unclaimed (no placeholder). **No Scoop bucket** (cargo-dist doesn't support it) тАФ Windows users use the PowerShell installer or `cargo binstall`; a hand-maintained bucket is a later, only-on-demand item.
- **Signing:** releases are **not OS-code-signed**; each artifact ships **keyless GitHub build-provenance attestations + SHA-256 checksums** instead (verify with `gh attestation verify <file> --repo Costroid/costroid`). The trade-off: first run shows a macOS "unidentified developer" / Windows SmartScreen prompt. To enable signing later (paid, and just config toggles on the existing pipeline): macOS notarization (Apple Developer ID, ~$99/yr) + Windows Authenticode (EV cert with HSM/cloud signing, ~$200тАУ300/yr), then cargo-dist's signing config + the matching secrets. Revisit only if those prompts become a real adoption problem.

- Don't literally rewrite from zero тАФ **port** the validated cost core; it's the trust asset and the bug-trap you already escaped.
- The tray/taskbar is the **planned last surface** (egui/eframe + `tray-icon`, not Tauri; PRODUCT-PLAN.md Step 6), after the CLI core; note "cross-platform terminal toolbar" still means the status-line integration, *not* the taskbar (most terminals have no toolbar to host).
- Pricing lives inside `costroid-core`, never a top-level directory тАФ cargo won't package it otherwise.
- Your live-quota fragility risk is the **Claude `statusLine` `rate_limits` field** (absent, poisoned, *and* false-in-range тАФ needs sanitize **+** local cross-check, ┬з9.2); it must degrade to "unavailable"/"unverified," never a confident wrong number. Cursor has no live source at all (no session reuse тАФ ┬з8), so its quota is simply "unavailable."
- Recommendation = frontier + your position, not per-task prescriptions; DeepSWE over CursorBench on neutrality.
- The "bigger picture total" is per-lane (API $ / subscription-equivalent $ / quota %), never one merged number.
- Sparkline is hand-rasterized (no Ratatui `Canvas`) тАФ one styling path feeds both the TUI and `--plain`; `NO_COLOR` drops ANSI styling only (braille fill stays glyph-distinct тАФ never color alone; ASCII is the braille-incapable fallback). Both **settled** (mechanics in ┬з7).
- Iterate the existing crates (published through 0.3.0; the 0.4.0 ladder adds `costroid-connect` тАФ T10b); don't orphan the claimed names.
- **Opus real-log quirk:** costroid runs ~0.08% under ccusage on opus totals тАФ isolated to re-logged sub-agent (sidechain) cache-read de-dup; mainline matches to the cent. Benign methodology difference, Claude parser unchanged (┬з9.1); the invoice is ground truth (Phase 2+).
- **WSL Windows-root discovery (fixed):** under WSL with `USERPROFILE` unset, costroid scans `/mnt/c/Users/*` for profiles holding `.claude`/`.config/claude`/`.codex`/`.cursor` and merges them тАФ and Codex discovery now merges all roots (was first-root-wins), with session-level cross-root dedup. Residual behaviors to know: a *set* `USERPROFILE` (even empty) is explicit тЖТ no scan; the scan is `/mnt/c` only (other drives need the env knobs); an empty scan falls back to the legacy `/mnt/c/Users/$USER` guess; it's evidence-based, so it includes *any* Windows profile with logs (e.g. a sandbox profile).
- **Claude live quota is sanctioned, not reverse-engineered (verified 2026-06-05).** Source = Claude Code's documented `statusLine` hook (the `rate_limits` block: `five_hour`/`seven_day` тЖТ `used_percentage` + `resets_at`; Pro/Max, after the first API response; zero API tokens), captured via the paste-in snippet or by being the statusline (┬з8) тАФ never by reusing Claude Code's token. So Costroid shows the **real** 5h/7d numbers with no account/ToS risk тАФ the opposite of codexbar. The field is **buggy** (┬з9.2): **absent** for API keys; **poisoned** (epoch in `used_percentage`); shows **impossible (900%)** and **false-in-range (a flat 100% with no throttling)** values тАФ so Costroid **sanitizes** (`>100` / `==resets_at`, at parse time) *and* **cross-checks** a high reading against local volume, demoting it to "unverified" rather than trusting it (both built in T4, ┬з9.2). It's **push-only** (label "as of HH:MM / stale"; `refreshInterval` re-renders but does **not** freshen quota тАФ the field only moves on a new API response), reads **low** if quota was spent on claude.ai chat (disclose, not fixable), and omits the **per-model weekly windows** тАФ so lead with the overall 7d % as the measurable number (not the binding cap; for a ~97%-Opus user the Opus weekly may bind first); the companion Opus-specific render тАФ Opus 7d volume/value (a local-log approximation) with the cap-% **unavailable**, plus the 7d-window-may-not-be-the-throttling-window disclosure тАФ is **design intent, not yet shipped** (┬з8). Parse `resets_at` defensively (seen as both epoch seconds and ISO across versions). This **supersedes** the `/api/oauth/usage` path for Claude; that path is dropped entirely, and there is no session-reuse plan for any provider (Cursor included тАФ its live quota is discovery-gated, ┬з8).
- **Cursor is detection-only, and stays that way until a sanctioned source exists** (no local data, no sanctioned hook; its usage/quota are served live server-side, and Costroid never reuses a session to fetch them). Phase 1 shows Cursor present + selected model, **beta**, usage/quota *"unavailable тАФ no sanctioned source."* Live Cursor quota is **discovery-gated (PRODUCT-PLAN.md ┬з8)** тАФ pursued only if Cursor publishes a documented API or first-party OAuth, never via session reuse against `api2.cursor.sh`. The quota *shape* is already resolved and supported: Cursor's paid plans are a **monthly, dollar-denominated credit pool + usage-based overage** (a `BillingCycle`, spend-$ window), and the **daily token window is the free-tier rate-limit** (not "daily, spend-$"); the generalized model (┬з6, PRODUCT-PLAN.md ┬з2a) supplies the `Monthly`/`BillingCycle` kinds and the `Spend` measure that fit it (landed in T2). Only a sanctioned *source* is missing. Cursor spend is a *spend* window, never summed into the cost lanes.
- **The two benchmarks have different weaknesses (verified 2026-06-04).** DeepSWE (updated 2026-05-30) is a **neutral** source, but its avg cost/task prices input at the cache-*miss* rate тАФ a ~5├Ч overstatement vs cache-correct pricing тАФ and scores on the neutral `mini-swe-agent` harness, *not* Claude Code/Codex/Cursor (the harness swings scores ~┬▒10 pts). CursorBench v3.1 (2026-05-18, `cursor.com/cursorbench`) is the opposite: a **vendor** benchmark (Cursor's own private, non-reproducible tasks, showcases Composer), but its cost is **cache-aware** тАФ computed from each model's published per-token pricing including cache reads тАФ so its $/task is methodologically sound; the caveat there is provenance, not cost. So step 4 takes the **quality/score** axis from both (DeepSWE primary, CursorBench corroborating) and the **dollar** axis from Costroid's own cache-correct cost (a task-average isn't the user's bill); un-benchmarked models (e.g. opus-4.8) show as gaps, never guessed (┬з9.3). Verified figures (all exact): DeepSWE avg cost/task (cache-miss-priced) gpt-5.5 70% @ $6.61, opus-4.7 54% @ $18.19, sonnet-4.6 32% @ $5.52; CursorBench (cache-aware) composer-2.5 63.2% @ $0.55, opus-4.7 Max 64.8% @ $11.02, gpt-5.5 64.3% @ $4.37.
